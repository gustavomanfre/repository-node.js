Reglas de nomenclatura

Nombre de variables

Reglas para nombrar una variable en JavaScript:

  • Los nombres de variables pueden contener letras, números, _ (underscore) y signo de dólar ($).
  • No pueden comenzar con un número.
  • Deben comenzar en minúsculas.
  • Los nombres son case sensitive (no es lo mismo A que a).

Si bien existe flexibilidad, en la actualidad, se recomienda crear variables con el formato camelCase. Este formato representa una estructura fiel a la que adopta JavaScript como lenguaje de programación. Veamos un ejemplo en el siguiente slide.

En JavaScript se estila declarar cada variable o constante utilizando una palabra, en minúsculas, que describa el tipo de datos que almacena. Es posible usar dos o más palabras, en aquellos casos donde una sola no alcance para describir el valor que se almacenará.

Cuando se combinen palabras usando el formato camelCase se debe contemplar que, cada palabra adicional tenga su primera inicial en mayúsculas.

Ejemplos:

  • userName (válido, usa camelCase)
  • totalAmount (válido, describe el contenido)
  • $price (válido, usa $)
  • 1value (inválido, comienza con número)
  • getData (inválido, usa verbo, más adecuado para funciones)
  • caña (inválido, contiene ñ)

Evitar la declaración de variables que incluyan:

  • Verbos en su nombre.
  • Uso de letras con tilde, diéresis, o ñuflos (letra ñ).
  • Primera inicial en mayúsculas (se reserva para la creación de objetos).

Nombre de proyectos

Reglas para nombrar un proyecto:

  • Deben ser cortos, descriptivos y únicos.
  • Usar solo letras, números y guiones (-).
  • No deben comenzar con números.
  • Se recomienda usar kebab-case (palabras separadas por guiones).
  • Evitar espacios y caracteres especiales.

Ejemplos:

  • task-manager (válido, corto y descriptivo)
  • blog-platform (válido, usa kebab-case)
  • 2023-project (inválido, comienza con número)
  • user_data (inválido, usa underscore en lugar de guiones)
  • my project (inválido, contiene espacio)

Nombre de archivos

Reglas para nombrar un archivo:

  • Usar letras minúsculas y números.
  • Separar palabras con guiones (-), siguiendo kebab-case.
  • No comenzar con números.
  • Evitar espacios y caracteres especiales.
  • Incluir extensión adecuada (ej. .js, .html).

Ejemplos:

  • index.html (válido, simple y con extensión)
  • user-profile.js (válido, usa kebab-case)
  • style-main.css (válido, descriptivo)
  • 1script.js (inválido, comienza con número)
  • user_data.js (inválido, usa underscore)
  • my file.js (inválido, contiene espacio)

Nombre de funciones

Reglas para nombrar una función:

  • Deben comenzar en minúsculas y usar camelCase.
  • Deben ser verbos o expresiones que describan la acción (ej. getData).
  • No usar guiones ni números al inicio.
  • Evitar tildes, diéresis o ñ.

Ejemplos:

  • calculateTotal (válido, describe acción)
  • fetchUserData (válido, usa camelCase)
  • updateProfile (válido, claro y descriptivo)
  • get_data (inválido, usa guiones)
  • 1startProcess (inválido, comienza con número)
  • procesarÁrea (inválido, contiene tilde)

Nombre de clases

Reglas para nombrar una clase:

  • Deben comenzar con mayúscula y usar PascalCase.
  • Deben ser nombres de sustantivos que representen el objeto (ej. UserProfile).
  • No usar guiones ni números al inicio.
  • Evitar tildes, diéresis o ñ.

Ejemplos:

  • UserProfile (válido, usa PascalCase)
  • DatabaseConnection (válido, describe el objeto)
  • ShoppingCart (válido, claro y descriptivo)
  • user-profile (inválido, usa guiones)
  • 1Product (inválido, comienza con número)
  • CiudadPequeña (inválido, contiene ñ)
