Explicación Detallada con Ejemplos

string
Representa una secuencia de caracteres. En JavaScript, los strings son inmutables y pueden crearse con comillas simples, dobles o backticks (ES6).
// Ejemplos básicos
let nombre = "Juan";
let apellido = 'Pérez';
let saludo = `Hola ${nombre}`; // ES6

typeof nombre; // "string"
// Métodos comunes
nombre.length; // 4
nombre.toUpperCase(); // "JUAN"
nombre.charAt(0); // "J"
nombre.includes("ua"); // true
number
Representa tanto números enteros como decimales. JavaScript usa punto flotante de doble precisión (64 bits) para todos los números.
// Números válidos
let entero = 42;
let decimal = 3.14159;
let negativo = -17;
let cientifico = 1.5e6; // 1500000

typeof entero; // "number"
// Valores especiales
let infinito = Infinity;
let noEsNumero = NaN;
0 / 0; // NaN
1 / 0; // Infinity

isNaN(NaN); // true

⚠️ Importante:

NaN NO es un tipo primitivo separado, es un valor especial del tipo number que significa "Not a Number".

boolean
Representa valores lógicos: verdadero (true) o falso (false). Se usa principalmente en condiciones y operaciones lógicas.
// Valores booleanos
let esVerdadero = true;
let esFalso = false;

typeof esVerdadero; // "boolean"

!true; // false
!false; // true
// Conversión implícita
Boolean(""); // false
Boolean("texto"); // true
Boolean(0); // false
Boolean(42); // true
Boolean(null); // false
undefined
Representa una variable que ha sido declarada pero no se le ha asignado un valor, o un valor que no existe.
// Casos de undefined
let sinValor;
console.log(sinValor); // undefined

let obj = {};
obj.propiedadInexistente; // undefined

typeof undefined; // "undefined"
// Función sin return
function sinReturn() {
console.log("Hola");
}

let resultado = sinReturn();
console.log(resultado); // undefined
null
Representa la ausencia intencional de cualquier valor de objeto. Es un valor primitivo que significa "vacío" o "sin valor".
// Uso de null
let valorNulo = null;
console.log(valorNulo); // null

null == undefined; // true
null === undefined; // false
// Bug histórico
typeof null; // "object" ⚠️
// Esto es un bug histórico
// null ES un primitivo

null instanceof Object; // false

⚠️ Bug Histórico:

typeof null devuelve "object", pero null ES un tipo primitivo. Este es un bug conocido que se mantiene por compatibilidad.

symbol
ES6/ES2015
Introduce un nuevo tipo primitivo que representa un identificador único. Se usa principalmente para crear propiedades de objeto que no colisionen.
// Creación de symbols
let sym1 = Symbol();
let sym2 = Symbol("descripcion");
let sym3 = Symbol("descripcion");

sym2 === sym3; // false
typeof sym1; // "symbol"
// Uso en objetos
let obj = {};
let propiedadUnica = Symbol("prop");

obj[propiedadUnica] = "valor secreto";
obj.propNormal = "valor normal";

Object.keys(obj); // ["propNormal"]
bigint
ES2020
Permite representar números enteros de precisión arbitraria, superando la limitación de Number.MAX_SAFE_INTEGER.
// Creación de BigInt
let numeroGrande = 123456789012345678901234567890n;
let otraForma = BigInt("123456789012345678901234567890");

typeof numeroGrande; // "bigint"
// Limitaciones
Number.MAX_SAFE_INTEGER; // 9007199254740991
let seguro = 9007199254740991n;

// Error: no se puede mezclar
// 10n + 5; // TypeError
10n + 5n; // 15n ✓
